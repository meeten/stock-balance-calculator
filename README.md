# Stock Balance Calculator

Kotlin CLI-приложение для обработки операций с товарами и расчета текущих складских остатков.

Программа принимает CSV-файл с операциями поступления и продажи товаров, выполняет их последовательно и формирует итоговый CSV-файл с актуальными остатками.

## Stack

- Kotlin
- Gradle
- JUnit 5

## Input CSV format

Входной CSV-файл содержит последовательность операций двух типов.

Разделитель данных — `;`.

### Income (поступление товара)

Формат:

```text
<groupId>;<productId>;<count>
```

Пример:

```text
A;1;10
A;2;20
B;10;15
```

Операция увеличивает количество указанного товара.

Если товар с таким `productId` уже существует в группе, количество суммируется.

---

### Sale (продажа товара из группы)

Формат:

```text
<groupId>;<count>
```

Пример:

```text
A;15
B;30
```

Операция уменьшает количество товаров внутри указанной группы.

---

## Output CSV format

Выходной файл содержит текущие остатки товаров:

```text
<groupId>;<productId>;<count>
```

Пример:

```text
A;1;0
A;2;15
B;10;5
```

Результат сортируется по:

1. `groupId`
2. `productId`

---

## Business rules

### Продажа товаров

При продаже товар списывается в следующем порядке:

1. Сначала товар с наивысшим рангом.
2. Если его количества недостаточно, продажа продолжается следующим товаром.
3. Ранг товара определяется лексикографической сортировкой по `productId`.

Например:

```text
productId:
1
2
3
```

Товар `1` имеет наивысший ранг и будет списываться первым.

---

### Продажа больше доступного остатка

Если количество продажи превышает общий остаток группы:

- все доступные товары списываются до `0`;
- оставшаяся часть записывается как отрицательный остаток.

Так как в задании не указано, к какому именно товару привязывать отрицательный остаток, в данной реализации он записывается на товар с наивысшим рангом (`минимальный productId`).

Пример:

Начальный остаток:

```text
A;1;10
A;2;20
```

Продажа:

```text
A;50
```

Результат:

```text
A;1;-20
A;2;0
```

---

## Assumptions

В реализации зафиксированы следующие допущения:

- Продажа из несуществующей группы считается ошибочной операцией.
- В этом случае выбрасывается `IllegalArgumentException`.
- Пустые строки в CSV-файле игнорируются.
- Пробелы вокруг значений CSV удаляются при чтении.

---

### Run

Linux / macOS:

```bash
./gradlew run --args="input.csv output.csv"
```

Windows (PowerShell):

```powershell
./gradlew run --args="input.csv output.csv"
```

Windows (Command Prompt):

```cmd
gradlew.bat run --args="input.csv output.csv"
```

Где:

- `input.csv` — входной CSV-файл с операциями;
- `output.csv` — файл, который будет создан с итоговыми остатками.

> [!IMPORTANT]
> Поместите файл `input.csv` в корневой каталог проекта или укажите относительный либо абсолютный путь к нему.

После выполнения появится файл:

```text
output.csv
```

с рассчитанными остатками.

## Tests

Добавлены unit-тесты, покрывающие основные сценарии:

- добавление товара;
- повторное поступление одного товара;
- обработку множества операций поступления;
- продажу товара;
- частичную продажу товара;
- продажу нескольких товаров внутри группы;
- списание товаров согласно рангу;
- формирование отрицательного остатка;
- восстановление остатка после отрицательного значения;
- обработку продажи из несуществующей группы.

Запуск тестов:

Linux / macOS:

```bash
./gradlew test
```

Windows:

```bash
gradlew.bat test
```
