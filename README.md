# Stock Balance Calculator

Kotlin CLI-приложение для обработки операций с товарами и расчета текущих остатков на складе.

Программа принимает CSV-файл с операциями поступления и продажи товаров и формирует итоговый CSV-файл с актуальными остатками.

## Stack

- Kotlin
- JUnit 5
- Gradle

## Input CSV format

Входной файл содержит операции двух типов.

### Income (поступление товара)

Формат:

```text
<groupId>;<productId>;<count>
```

Пример:

```csv
A;1;10
A;2;20
B;10;15
B;20;25
C;100;50
A;1;5
B;10;10
```

Операция увеличивает количество указанного товара.

### Sale (продажа товара из группы)

Формат:

```text
<groupId>;<count>
```

Пример:

```csv
A;15
B;30
C;60
A;100
```

Операция уменьшает количество товаров в указанной группе.

### Example input.csv

Полный пример входного файла:

```csv
A;1;10
A;2;20
B;10;15
B;20;25
C;100;50
A;1;5
B;10;10
A;15
B;30
C;60
A;100
```

В данном примере:

- первые 7 строк — операции поступления товаров;
- последние 4 строки — операции продажи товаров.

## Output CSV format

Выходной файл содержит актуальные остатки товаров:

```text
<groupId>;<productId>;<count>
```

Пример:

```csv
A;1;-80
A;2;0
B;10;0
B;20;20
C;100;-10
```

## Running

### Using Gradle

Запуск:

```bash
./gradlew run --args="input.csv output.csv"
```

Для Windows:

```bash
gradlew.bat run --args="input.csv output.csv"
```

Где:

- `input.csv` — входной файл с операциями;
- `output.csv` — файл с результатом обработки.

После выполнения будет создан файл `output.csv`.

## Business rules

- При поступлении товара количество увеличивается.
- При продаже сначала списывается товар с наивысшим рангом.
- Ранг товара определяется лексикографической сортировкой по `productId`.
- Меньшее значение `productId` считается более высоким рангом.
- Если товара с наивысшим рангом недостаточно, продажа продолжается по следующему товару.
- Если количество продажи превышает доступный остаток группы:
  - все доступные товары списываются;
  - оставшаяся часть записывается как отрицательный остаток.

## Assumptions

В задании не указано, к какому товару необходимо привязать отрицательный остаток, если продажа превышает общий остаток группы.

В данной реализации отрицательный остаток записывается на товар с наивысшим рангом (`минимальный productId` при лексикографической сортировке).

Пример:

До продажи:

```text
A;1;0
A;2;0
```

Операция продажи:

```text
A;10
```

Результат:

```text
A;1;-10
A;2;0
```

Также принято решение:

- продажа из несуществующей группы считается ошибочной операцией;
- такая операция приводит к `IllegalArgumentException`.

## Tests

Добавлены unit-тесты, покрывающие:

- добавление товара;
- повторное поступление товара;
- хранение нескольких товаров;
- продажу товара;
- частичную продажу;
- продажу нескольких товаров внутри группы;
- списание по рангу товара;
- отрицательный остаток;
- восстановление остатка после отрицательного значения;
- обработку продажи из несуществующей группы.
